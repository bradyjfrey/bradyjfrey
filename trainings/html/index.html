HTML or XHTML

What is the difference?

  • HTML is just HTML
  • XHTML is a XML language, coded to work with HTML. It is meant to ease the transition into XML for the world and was originally intended to be the successor to HTML. However, XHTML has found to be fantastic for advanced applications, whereby HTML still thrives for web design: the W3 has recently adopted an HTML 5 from the public working group called WHATWG.
  • XHTML looks just like HTML, with the following exceptions:
    • You must use an XHTML/XML document type
    • No uppercase tags are allowed
    • All tags must be closed. Tags that do not have a closing element get a />
  • XHTML allows you to integrate techniques into advanced applications - for example, all the preferences in OS X are XML, all the new file formats for Microsoft Office are a branch of XML, Dreamweaver is almost completely designed as XML files, etc.
  • One Issue: IE still doesn't support 'full' XHTML

What are Standards?

Standards are when webpages use proper HTML (or XHTML) with CSS and other languages to produce websites that follow modern, expert, guidelines.

Websites that use standards follow these 3 steps:

  1. They produce Valid Code.
  2. Their content is semantic.
  3. They separate their Content, from their Presentation, from their Interaction.

Valid Code

Valid code is HTML (or XHTML) that uses a proper Doctype, and has been run through a validator to fix current and future problems.

What does Valid Code do for me?

Many search engines cannot catalog sites with major validation errors; much less semantic code improves search engine ranking of content with major engines like Google. Your code will render consistently and faster (errors have to be negotiated in a browser), which results in higher retention of clientele. You'll have no fear of outdated sites, as you do now: valid code is 'future proof' by web standards.

Failure to make your code valid harms your clients business potential through customer reach and costly redesigns. Blame on templates, dynamic code, editors and the social norm are common copouts for our own lack of initiative to learn and improve.

Validators:

  • HTML: W3C Validator
  • CSS: Jigsaw CSS validator

Semantic Code

The intent of Semantic Code is that each element of content on a page is surrounded by tags that convey meaning and value to the user, to search engines, and to your computer.

Semantic Code will rank you higher, since search engines will know that important elements defining the document are in headers, paragraphs are content, lists are lists, and more. Ignore how HTML looks when written semantically, you will override all of this with your CSS.

Separation of Content, Presentation, & Interaction

Separating your content into HTML (or dynamic languages that output HTML), design in CSS, and Interactivity in Javascript allows your code to be cached in the users computer for faster load time, and is easier to find, adjust, and reuse your code overtime.

Handicap Accessibility for the Web

If you followed the above advice, you're sites will most likely be accessible to all visitors. You've gained an increase (and heavily internet purchasing demographic) in market share, saved your company potential money in lawsuits, and are in general a nice person.

The basic structure of HTML

HTML consists of a Doctype, a Head with meta information, and Body with your live content.

Header tags, paragraphs, lists, cite, emphasis and strength tags define your web pages content.

The Table

Because the old web did not have CSS that we could use for design, many developers used tables for design purposes. This is no longer the case, and hasn't been since IE 5+. Tables should be used for tabular data (such as calendars or charts), and nothing more.

Why table for layout is stupid?

Class & ID

Classes & ID's are used to target specific HTML tags for use in CSS and Javascript, but they are not always needed (besides it's best to avoid using them often).

Div & Span

A Div serves as a page Divider, but has no semantic meaning. A span also has no semantic meaning, and is used simply when you need a tag that doesn't effect semantics. Use these only as design tools, since excessive use can do more harm than good to your code.

CSS

CSS is the language of designers; it allows you to convert that dreary HTML into a beautiful design. The power of CSS is it's cascading and caching benefits across browsers, for easier and faster downloads. It also lets you update look without affecting content.

CSS Zen Garden, an example of CSS in action.

Adding CSS to your HTML

You can add CSS in 3 ways:

  1. External: Link or import your CSS. This is the best and most powerful method.
  2. Internal: Put your CSS in the head tag of your HTML page. This loses much of the benefits of CSS cascading and separation, but is much cleaner than...
  3. Inline: Add CSS directly to a tag. This is no different than older, deprecated methods of styling and is not recommended unless you absolutely need too.

Box model & positioning

Every HTML element is contained within a box. This square box is controlled by CSS with width/height, padding, borders, margins, and a variations of design treatments.

You can also position your elements anywhere on the page either by the flow of the document, or overriding that with CSS:

  1. Position: Relative -- Keeps the box in the document, and lets you control how it's moved
  2. Position: Absolute -- takes the box out of the document, and lets you control how it's moved
  3. Position: Fixed -- takes the box out of the document, and fixes it somewhere on the window

The Failure of Flash

Flash is a closed source, proprietary language by Adobe (formerly Macromedia), so you are limited by the companies vision of what you need. It's design by nature is like an image - no searchable content, no way to bookmark individual pages, all contained in a package that has little accessibility. Flash is not a replacement for HTML, it is a tool to compliment HTML and should be used sparingly.

The Success of Flash

The scripting power of flash, coupled with the power over all aspects of the experience in flash, has made it an excellent contender for web applications only marred by it's limited community voice and global reach as noted above.

What's XML?

XML, as noted briefly above, is a text language devoid of any limitation. It can be used to store data or interpret rules with other languages. It's simple, free, universal, and used everywhere from desktops to websites depending on it's need. Microsoft has marketed aggressively to skew XML in it's favor.

SEO or Search Engine Optimization

You can improve your search engine ranking by writing semantic, valid code - gaining increased traffic and linking between other sites - and not too much else. Firms selling SEO are doing exactly this, including reaching out to their large networks to add you as a member - any other techniques are snake oil.

Black Hat techniques such as hiding fake words in the site or doing questionable linking can get you banned on Search Engines forever.

Design for the Web

The almost immediate mistake of any seasoned designer is to turn web design into print design. The web is not print: you cannot completely control the dimensions of your media, you have no control over how they view your media, and you cannot force it in their hands.

  • Limited to 8 (yes 8) fonts that for sure people have, and you can't embed fonts (save a little known flash and javascript trick)
  • Email browsers are 7 years behind the times

Color on the internet

While the color space for monitors is a larger gamut (can display a larger amount of colors from the device), you cannot control what type of monitor they have. Expect inconsistencies in how your design is viewed, you'll have to live with it, even with ICC profiles.

Fonts on the Internet

Because of licensing restrictions and conflicts, there are only 8 guaranteed fonts you are sure all users will have worldwide (ouch). You can use images, flash, or javascript techniques to replace this text, but your user loses the ability to fully interact with that text.

PNG, GIF, JPEG

  1. GIF = exact colors limited to 256 at the most, great compression, does 0% or 100% transparency only -- suited for vector style graphics with simple transparency, or for animation
  2. JPEG = Color limited to only the monitors output - no transparency - great compression but can saturate vector style graphics, suited for photos
  3. PNG = all the great stuff of a gif and a jpeg combined (minus the animation, but there is a png format for that just not widespread), full transparency levels, full color and limited colors, all exact. Large photos can result in larger than needed file sizes.

...but, IE 6 and down do not handle png transparencies properly.
