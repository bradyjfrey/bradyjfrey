Modern Web Design, on borrowed time

WYSIWYG vs. Hand Coding

For the most part, WYSIWYG editors have improved over the years, but they still cater to their companies update policy, and will never fully understand the human value of what 'semantics' are to content. Save yourself time by using a hand coding editor as you improve, rather than fixing the code these editors dish out.

Defining Standards

Standards are when webpages use proper HTML (or XHTML) with CSS and other languages to produce websites that follow modern, professional, guidelines.

Websites that use standards follow these 3 steps:

  1. They produce Valid Code.
  2. Their content is semantic.
  3. They separate their Content, from their Presentation, from their Interaction.

Standards: Valid Code

Valid code is HTML (or XHTML) that uses a proper Doctype, and has been run through a validator.

Search engines cannot catalog sites with major validation errors; much less semantic code improves ranking. Your code will render consistently & faster (errors have to be negotiated in a browser). You'll have no fear of your sites becoming outdated: valid code is 'future proof' by web standards.

Failure to make your code valid limits your clients business through customer reach and costly redesigns. Blame on templates, dynamic code, editors and the social norm are common copouts for our own lack of initiative to learn and improve.

  • HTML: W3C Validator
  • CSS: Jigsaw CSS validator

Standards: Semantic Code

The intent of Semantic Code is that each element of content on a page is surrounded by tags that convey meaning and value to the user, to search engines, and to your computer.

Semantic Code will rank you higher, since search engines will know that important elements defining the document are in headers, paragraphs are content, lists are lists, and more. Ignore how HTML looks when written semantically, you will override all of this with your CSS.

Standards: Separation

Separating your content into HTML (or dynamic languages that output HTML), Presentation in CSS, and Interactivity in Javascript (or other) allows your code to be cached in the users computer for faster load time, and is easier to find, adjust, and reuse your code overtime.

Standards: Handicap Accessibility

If you followed the above advice, you're sites will most likely be accessible to all visitors. You've gained an increase (and heavily internet purchasing demographic) in market share, saved your company potential money in lawsuits, and a good human being. Not to mention, Google reads a webpage as a blind user would.

HTML: Just Say Yes

XHTML was slated to replace HTML... but misguided by application design, it no longer serves our web community as most designers need. HTML 5 is the future, HTML 4 is still King.

  • XHTML is an XML language, coded to work with HTML. It is meant to ease the transition into XML for the world and was originally intended to be the successor to HTML. However, XHTML has found to be fantastic for advanced applications, whereby HTML still thrives for web design: the W3 has recently adopted an HTML 5 from the public working group called WHATWG.
  • XHTML looks just like HTML, with the following exceptions:
    • You must use an XHTML/XML document type
    • No uppercase tags are allowed
    • All tags must be closed. Tags that do not have a closing element get a />

HTML: The Structure

Every webpage is created with the same base tags:

  1. Doctype
  2. HTML Tag
  3. Head Tag
  4. Body Tag

Your tags should be written in lowercase, without typos. Some tags require an opening and closing part like so:

<p>Open on the left, close on the right</p>

HTML: The Doctype

The problem with web development is the same as it's benefit: speed. Changes come fast - but the sheer scale of how many websites out there means any change can conflict with millions+ of websites. Salvation is in a Doctype.

Doctypes tell browsers what version of HTML you are using, so that old code can be backward compatible, and new code can be used.

Most folks will use HTML 4.01 Strict as their Doctype, but there are many to choose from, depending on your application!

Recommended Reading: Fix your site with the right Doctype

HTML: The HTML Tag

Now that you have a doctype, add an 'opening' and 'closing' HTML tag beneath it:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>

</html>

HTML: The Head Tag

Inside these HTML tags, you can put an opening and closing Head Tag:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>

  </head>
</html>

HTML: The Body Tag

Underneath the Head Tag, you need to add an opening and closing Body Tag to finish your webpage:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>
  </head>
  <body>

  </body>
</html>

HTML: The Title Tag

Inside the Head Tags, you can add a page title:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>
    <title>Mac IT</title>
  </head>
  <body>

  </body>
</html>

HTML: Metas

Inside the Head Tags, you can add a page title:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<html>
  <head>
    <title>Mac IT</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
  </head>
  <body>

  </body>
</html>

HTML: Headers

Now that the structure is defined, you can add Content to your page. Headers are from h1 - h6 as a way of highlighting important topics of your page inside the Body Tags:

  <body>
    <h1>Mac IT</h1>
    <h2>Control, Alt, Delete not included</h2>
  </body>

Headers don't have to go in order, you can move put h6's in front of h3's, and so forth.

HTML: Paragraphs

Exactly what it means, a paragraph of content. If you want a line break, just end and start a new paragraph, no need for <br> tags!

  <body>
    <h1>Mac IT</h1>
    <h2>Control, Alt, Delete not included</h2>
    <p>Another paragraph of information.</p>

  </body>

HTML: Lists

There are 3 types of lists: Unordered List, Ordered List, and Definition List:

    <h1>Mac IT</h1>
    <h2>Control, Alt, Delete not included</h2>
    <ul>
     <li>Home</li>
     <li>About</li>
     <li>Services</li>
     <li>Contact</li>
    </ul>
    <p>Another paragraph of information.</p>

HTML: Anchors (or Links)

Links are used to point text to another site, file, page or email but putting an address in 'href'. Wrap 'real words' not 'click here', like so:

<h1><a href="http://macit.com">Mac IT</a></h1>
<p><a href="http://google.com">Google</a> is the champ.</p>

Would look like:

Mac IT

Google is the champ.

HTML: Images

Images were never intended to be a tag by itself, and it's invention by Netscape caused some problems. Nevertheless, they're here to stay and easy to add:

<img src="photo.jpg" width="50" height="20" alt="Sunset Photo" >
  • This tag does not have a closing tag.
  • Like the Anchor Tag, you put the location of the file (it could be on the internet too, http://bradyjfrey.com/photo.jpg).
  • For width and height, you put the pixel dimension in the corresponding field.
  • Alt is where you put alternative text for search engines and handicap users to know what you've linked; if there's no good description, leave alt="" empty.

HTML: The Controversial Div

A Div serves as a page Divider, but has no semantic meaning. Use these only as structure tools: excessive use can do more harm than good to your code.

   <div>
    <h1>Mac IT</h1>
    <h2>Control, Alt, Delete not included</h2>
   </div>

HTML: The Controversial Table

Because the old web did not have CSS that we could use for design, many developers used tables for design purposes. This is no longer the case, and hasn't been since IE 5+. Tables should be used for tabular data (such as calendars or charts), and nothing more.

Why table for layout is stupid?

HTML: Class & ID

Classes & ID's are used to target specific HTML tags for use in CSS and Javascript, but they are not always needed (besides it's best to avoid using them often). You may call an ID once per page, classes can be called multiple times per page.

   <div id="header">
    <h1>Mac IT</h1>
    <h2>Control, Alt, Delete not included</h2>
   </div>

CSS: Design for the web

The almost immediate mistake of any seasoned designer is to turn web design into print design. The web is not print: you cannot completely control the dimensions of your media, you have no control over how they view your media, and you cannot force it in their hands.

Your designs should be contained in the browser, but not manipulate the browser application.

CSS: Adding to HTML

You can add CSS in 3 ways:

  1. External: Link or import your CSS. This is the best and most powerful method.
    <link rel="stylesheet" type="text/css" media="screen,projection" href="style.css">
  2. Internal: Put your CSS in the head tag of your HTML page. This loses the benefits of cache...
    <style type="text/css">
h1 {color: blue;font-size: 24px;}
</style>
  3. Inline: Add CSS directly to a tag. This is no different than older, deprecated methods of styling and is not recommended unless you absolutely need too.
    <h1 style="color: blue;font-size: 24px;">Mac IT</h1>

CSS: Syntax

Writing CSS is simple. You can write it by calling a tag, an ID, a class, or even tags within tags. Commands are listed inside {}. To call an HTML tag:

h1 {color: blue;}

or an ID:

#myid {color: blue;}

or a class:

.myid {color: blue;}

Possibly even an h1 inside a div named brady:

#brady h1 {color: blue;}

CSS: The Box Model

Every HTML element has an invisible space around it called a 'Box'. We can provide width, height, padding, margins, borders, background images and colors to this Box using CSS.

Here are examples of the box model in drawing.

CSS: Padding & Margins

Padding is outside the width of a box, margin is outside the padding. If we had:

h1 {width: 200px;padding-left: 20px;padding-right: 20px;}

...our total width is 240px (width + left padding + right padding). If we added margins, it would be even larger!

CSS: Borders

Borders sit between padding & margin, and increase the overall dimension. If we had:

h1 {width: 200px;padding-left: 20px;padding-right: 20px;
   border-left: 1px solid black;border-right: 1px solid blue}

The total dimension is now 242px.

CSS: Floats

Objects such as images and divs, can be floated to the right or left; this allows text to flow around the object. If I had:

p img {float: right;}

Every image tag within a paragraph would float to the right side of the page, and the text would wrap around it, for example this is a float. Else it would look like this.

CSS: Typography

Since there are only 8 fonts we're sure each computer has, there's no real unanimous way (yet) to load up any unique fonts. For now, we list size, line-height, style and font family:

h1 {font-size: 14px;line-height: 21px;font-family: georgia,times,serif;}

CSS: Backgrounds & Color

Color can be applied to any background, type, or border. Colors can be written as Hex code, such as the #ffffff, or their RGB value, rgb(255, 255, 255), or in some cases just by saying 'white'. Colors are written:

h1 {color: white;background-color: black;border-color: pink;}

Background images are significantly more tricky: you can repeat them one way, both ways or not at all- you can place them anywhere within the Box, and you can put a color behind it:

h1 {background: black url(kirk-and-spock.jpg) no-repeat left center;}

CSS: Available Media

And you can do more than just on a screen... you can apply CSS specifically for the iPhone, for older browsers such as Internet Explorer, for print (print this presentation for example and see), even for audio feedback.

Design: How people read

Overwhelmingly, studies have shown that the vast majority of internet users scan text, rather than read full bits of content (like... right now). Structure your design and content around quick bites of information, unless it's a technical document for future growth (also... like right now).

Design: Fonts

Because of licensing restrictions and conflicts, there are only 8 guaranteed fonts you are sure all users will have worldwide (ouch). You can use images, flash, or javascript techniques to replace this text, but your user loses the ability to fully interact with that text.

Design: Color Limitations

While the color space for monitors is a larger gamut (can display a larger amount of colors from the device), you cannot control what type of monitor they have. Expect inconsistencies in how your design is viewed, you'll have to live with it, even with ICC profiles.

Design: Internet Explorer & Mail Clients

While Netscape Navigator (and their stepchild, Mozilla) openly embraced standards publicly & in development, Microsoft embraced it publicly, and did little development. The result was IE 6, riddled with insecurities and errors, held back the web for almost a decade.

Better yet, Mail Clients are only now beginning to implement the same rendering engines into their mail as their browser. Outlook 2010 is slated to even be a step back.

Design: Flash

Flash is a closed source, proprietary language by Adobe (formerly Macromedia), so you are limited by the companies vision of what you need. It's design by nature is like an image - no searchable content, no way to bookmark individual pages, all contained in a package that has little accessibility. Flash is not a replacement for HTML, it is a tool to compliment HTML and should be used sparingly.

...but the scripting power of flash, coupled with the power over all aspects of the experience in flash, has made it an excellent contender for web applications only marred by it's limited community voice and global reach as noted above.

Design: SEO

You can improve your search engine ranking by writing semantic, valid code - gaining increased traffic and linking between other sites - and not too much else. Firms selling SEO are doing exactly this, including reaching out to their large networks to add you as a member - any other techniques are snake oil.

Black Hat techniques such as hiding fake words in the site or doing questionable linking can get you banned on Search Engines forever.

Wordpress

Coding knowledge helps for a quality website, but there are applications such as Wordpress which come pre-built with proper standards and are easy to install. Here we'll finish up with Wordpress' famous 5 minute install. This installation assumes you're comfortable with Apache on Linux, Unix or OS X.

Wordpress has become the standard for blogging, and in some cases, a standard for CMS. Upgrades can be done in real time within the application, and there's a thriving developer community.

Wordpress: PHP on OS X

One unfortunate thing to point out is that, although OS X ships with a decent (and current) version of PHP, they have omitted additional libraries that benefit website developers. Most importantly, is the GD Library suite of image manipulation tools.

Since Apple hasn't yet ironed out a friendly development environment, the best place to get a quality version of PHP instead of rolling your own, is Mark Liyanage's PHP. This is the version Apple uses for their production servers as well.

Wordpress: Setup MySQL

Command-line into MySQL and create a database for you to use (keep note of the username, password, database name and the localhost name of your server).

Wordpress: 5 Minute Installation

  • Open the unzipped wordpress.org files you've download and rename wp-config-sample.php to wp-config.php.
  • Open wp-config.php and follow the directions to add your username, password, database, and secret phrase
  • Copy those files to your server, and launch your web address!

Wordpress: Customize

That's it. Wordpress will email you the administration password, and you can search around. Feel free to install new themes, or plugins from their Extend Site

Wordpress: Mu & BuddyPress

Wordpress also manages the official Wordpress Multi-User system (which allows you to maintain multiple sites and blogs from one admin), and the new BuddyPress (an open source, Facebook style system on your server).
